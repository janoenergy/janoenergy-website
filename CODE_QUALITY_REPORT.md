### 3.4 架构设计 🟡 (6/10)

#### 优点 ✅

1. **分层清晰**
   - app/ - 页面层
   - components/ - 组件层
   - lib/ - 工具层
   - services/ - 服务层

2. **路由设计合理**
   - 使用 Next.js App Router
   - 动态路由支持多语言

3. **状态管理**
   - React Context 管理主题
   - localStorage 持久化

#### 缺点 🔴

1. **类型定义分散**
   - `Project` 类型定义在多处
   - 应该统一在 `types/` 目录

2. **API 调用分散**
   - 直接在组件中 `fetch`
   - 应该统一封装在 `services/`

3. **图片管理混乱**
   - URL 散落在各处
   - 应该集中配置

4. **组件职责不清**
   - 部分组件过于庞大
   - 业务逻辑和 UI 混合

## 五、重构建议

### 5.1 目录结构优化

```
src/
├── app/                    # 页面层（保持简洁）
├── components/
│   ├── ui/                # 基础 UI 组件
│   ├── layout/            # 布局组件
│   ├── sections/          # 页面区块组件
│   └── features/          # 功能组件
├── hooks/                 # 自定义 Hooks
├── lib/                   # 工具函数
├── services/              # API 服务（需要加强）
├── types/                 # 全局类型定义
├── constants/             # 常量配置（新增）
│   ├── images.ts         # 图片 URL 配置
│   └── api.ts            # API 端点配置
└── utils/                 # 工具函数
```

### 5.2 代码规范

2. **函数行数限制**: 最大 50 行
3. **禁止 any**: 使用 `unknown` 或具体类型
4. **统一错误处理**: 使用 Error Boundary
