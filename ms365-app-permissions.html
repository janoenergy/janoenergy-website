⚠️ 重要：需要添加应用程序权限

当前权限类型不正确

你目前添加的所有权限都是"委托权限"（Delegated），这适用于用户登录场景。

但 OpenClaw 使用的是"客户端凭证"流程（Client Credentials），需要"应用程序权限"（Application）才能工作。

1 打开 Azure Portal 应用注册

点击下方按钮进入 API 权限配置页面：

🚀 打开 Azure Portal
2 点击"添加权限"按钮
📍 在 API 权限页面，找到并点击 "添加权限" 按钮
（位于"配置权限"标题下方）
3 选择权限类型 - 关键步骤！
⚠️ 必须选择"应用程序权限"，不是"委托权限"
  1. 点击 "Microsoft Graph"
  2. 选择 "应用程序权限"（Application permissions）
  3. ❌ 不要选"委托权限"（Delegated permissions）
📍 选择"应用程序权限"选项卡
4 搜索并添加以下权限

在搜索框中逐个搜索并勾选：

权限名称 功能
User.Read.All 读取所有用户的基本信息
Mail.ReadWrite 读写所有用户的邮件
Mail.Send 以任意用户身份发送邮件
Calendars.ReadWrite 读写所有用户的日历
Sites.Read.All 读取所有 SharePoint 站点
5 点击"添加权限"确认

勾选所有需要的权限后，点击底部的 "添加权限" 按钮

6 授予管理员同意 - 必须步骤！
⚠️ 应用程序权限必须获得管理员同意才能生效
  1. 回到 API 权限列表页面
  2. 点击顶部的 "授予管理员同意" 按钮
  3. 在弹出的对话框中点击 "是"
📍 点击"授予管理员同意"按钮后，权限状态会从"未授予"变为"已授予"
7 验证权限配置

完成后，你的权限列表应该包含以下应用程序权限：

权限 类型 状态
Application.Read.All 应用程序 ✅ 已授予
Files.ReadWrite.All 应用程序 ✅ 已授予
User.Read.All 应用程序 ✅ 已授予
Mail.ReadWrite 应用程序 ✅ 已授予
Mail.Send 应用程序 ✅ 已授予
Calendars.ReadWrite 应用程序 ✅ 已授予
Sites.Read.All 应用程序 ✅ 已授予

❓ 常见问题

Q: 为什么不能使用委托权限？

A: OpenClaw 作为后台服务运行，使用客户端凭证流程（Client Credentials）获取访问令牌，这种方式不支持委托权限，只能使用应用程序权限。

Q: 授予管理员同意时出现错误？

A: 只有 Azure AD 全局管理员或特权角色管理员才能授予应用程序权限的管理员同意。如果你没有权限，请联系你的 IT 管理员。
